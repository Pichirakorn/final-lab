## 3. สิ่งที่ได้ลงมือพัฒนาด้วยตนเอง

- เขียน API สำหรับ login ใน Auth Service
- การสร้าง JWT token หลังจาก login สำเร็จ
- การตรวจสอบ JWT token ใน middleware
- การทดสอบ API ด้วย Postman และ curl

นอกจากนี้ยังได้ช่วยตรวจสอบการเชื่อมต่อระหว่าง Auth Service และ Task Service เพื่อให้ระบบสามารถทำงานร่วมกันได้อย่างถูกต้อง

## 4. ปัญหาที่พบและวิธีการแก้ไข

### ปัญหา JWT Token ไม่สามารถตรวจสอบได้

สาเหตุเกิดจากการที่ คิดว่า ต้องวาง Token ใน Header บางครั้งได้บางครั้งไม่ได้

วิธีการแก้ไข  
ใส่ใน ข่อง Authorization เลือก Auth Type เป็น Bearer token แล้วใส่ Token เลย 


## 5. สิ่งที่ได้เรียนรู้จากงานนี้

- แนวคิดของ Microservices Architecture
- การใช้ Docker และ Docker Compose ในการจัดการหลาย services
- การใช้ HTTPS เพื่อเพิ่มความปลอดภัยของระบบ
- การใช้ JWT สำหรับ authentication
- การออกแบบ API และการทดสอบระบบด้วย Postman
