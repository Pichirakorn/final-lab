# INDIVIDUAL_REPORT
## 1. ข้อมูลผู้จัดทำ

ชื่อ: นายพิชิรกร ชาติปิระ
รหัสนักศึกษา: 67543210004-7

## 2. ส่วนที่รับผิดชอบ

- รับผิดชอบ Task Service
- รับผิดชอบ Log Service
- ตั้งค่า HTTPS และ Nginx Reverse Proxy
- จัดการฐานข้อมูลและ seed users สำหรับทดสอบ
- ออกแบบ architecture diagram ของระบบ

## 3. สิ่งที่ได้ลงมือพัฒนาด้วยตนเอง

- พัฒนา Task Service API สำหรับจัดการข้อมูล task เช่น การเพิ่ม แก้ไข และดึงข้อมูล task
- พัฒนา Log Service สำหรับบันทึกเหตุการณ์การทำงานของระบบ
- ตั้งค่า HTTPS ด้วย self-signed certificate เพื่อเพิ่มความปลอดภัยของระบบ
- ตั้งค่า Nginx Reverse Proxy เพื่อให้สามารถเรียกใช้งาน service ต่าง ๆ ผ่าน endpoint เดียว
- สร้าง seed users ในฐานข้อมูล สำหรับใช้ในการทดสอบระบบ
นอกจากนี้ยังได้ช่วยทดสอบการทำงานของระบบแบบ end-to-end เพื่อให้แน่ใจว่า frontend, services และ database สามารถทำงานร่วมกันได้อย่างถูกต้อง

## 4. ปัญหาที่พบและวิธีการแก้ไข
ปัญหา Service ไม่สามารถเชื่อมต่อกับฐานข้อมูลได้

- สาเหตุเกิดจากการตั้งค่า connection string ใน Docker Compose ไม่ถูกต้อง และมีการใช้ hostname ไม่ตรงกับชื่อ service ใน container

วิธีการแก้ไข
- ตรวจสอบไฟล์ docker-compose.yml และแก้ไข connection string ให้ใช้ชื่อ service ของ database แทนการใช้ localhost ทำให้ service สามารถเชื่อมต่อกับฐานข้อมูลได้ตามปกติ

## 5. สิ่งที่ได้เรียนรู้จากงานนี้

- การออกแบบระบบแบบ Microservices Architecture
- การสร้าง API สำหรับจัดการข้อมูลใน service ต่าง ๆ
- การตั้งค่า HTTPS และ Reverse Proxy ด้วย Nginx
- การจัดการ container ด้วย Docker และ Docker Compose
- การทำงานร่วมกันของหลาย services ภายในระบบเดียว

## 6. แนวทางที่ต้องการพัฒนาต่อใน Set 2

- เพิ่มระบบตรวจสอบสิทธิ์ของผู้ใช้ในแต่ละ service ให้มีความปลอดภัยมากขึ้น
- ปรับปรุงระบบ logging และ monitoring เพื่อให้สามารถตรวจสอบการทำงานของระบบได้ง่ายขึ้น
- ปรับปรุงโครงสร้างของ service ให้สามารถรองรับการขยายระบบในอนาคตได้
- เพิ่มการทดสอบระบบเพื่อให้มั่นใจว่าระบบสามารถทำงานได้อย่างถูกต้องในทุกส่วน
